必要なプログラムおよびライブラリ

  • libtool : 必須
  • aclocal : 必須
  • autoheader : 必須
  • automake : 必須
  • autoconf : 必須
  • make : 必須
  • g++ : 必須
  • g77 or gfortran :
  • cfortran :
  • libxml++ : 必須
  • doxygen : できればgraphvizも。
  • Berkeley DB v4 : 必須
    例えば Debian では libdb4.4, libdb4.4++, libdb4.4++, lindb4.4++-dev, db4.4-utilといったパッケージ になっています。
上記ライブラリ等があれば、一般的なLinuxディストリビューションで ライブラリをビルド可能だとおもいます。

configureスクリプトを作る　

この一連の作業はCVSからパッケージをcheckoutした直後、 またconfigure.acファイルを編集した後に行う必要があります。

準備

$ aclocal
$ libtoolize --force --copy                     libtoolの準備
$ autoheader                                    config.h.in 作成
$ automake --add-missing --copy --force-missing Makefile.in 作成
$ autoconf                                      configure 作成

configureスクリプトを実行する

開発する環境（システムバージョン、OSタイプ、ホスト等）が変わったとき、 configureスクリプト自身が更新された時に必要です。 
$ ./configure
これで、*.inというファイルから、 .inのとれたファイルが生成されます （例：Makefile.in→Makefile）。

*.inファイルが更新された場合、 *ファイルの生成には configureスクリプトではなく、config.statusを使います。 

$ ./config.status

makeを実行する

$ make

クラスリファレンスの作成

$ make doc
